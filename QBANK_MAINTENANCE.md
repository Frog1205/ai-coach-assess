# ATLAS 题库月度维护手册

> 每月 5-10 号花 30 分钟,把题库版本号 + 4-5 道前沿题更新进去,保证诊断结果跟得上 AI 行业演化。

---

## 1. 速查:每月要改的 3 个地方

打开 `website/index.html`,搜索这三处,各改一次:

| 位置 | 关键字搜索 | 改什么 |
|---|---|---|
| ① 版本号 | `QBANK_VERSION` | `'2026.05'` → `'2026.06'` |
| ② 更新日期 | `QBANK_UPDATED` | `'2026-05-15'` → 当月发布日 |
| ③ 题目数组 | `const QS = [` | 替换 4-5 道前沿题(每维度各 1) |

改完保存即可,无需重启、无需后端。历史用户的旧报告自动带 `qbank: '2026.05'` 标记,跨版本对比有提示。

---

## 2. 题目结构规范

每道题严格遵守:

```javascript
{ d: <维度索引 0-4>, q: '<问题正文,问号结尾>',
  a: [
    '<选项A · 完全不懂/没接触>',         // 0 分
    '<选项B · 听过/试过/浅尝>',          // 1 分
    '<选项C · 能用/会用/有基础体系>',      // 2 分
    '<选项D · 深度实践/有可复用资产>'      // 3 分
  ]
}
```

### 维度索引对照

| d | 维度 | 评估什么 |
|---|---|---|
| 0 | 工具认知 | 模型/工具的广度 + 选型判断力 |
| 1 | 提示词能力 | 表达精度 + 上下文构建 + 思维链 |
| 2 | 内容创作 | 文/图/视频/数字人的产出能力 |
| 3 | 自动化思维 | Agent / 自动化任务 / 替代手动操作 |
| 4 | 工作流整合 | 体系化 / 团队化 / 业务嵌入深度 |

### 选项设计四档梯度(必须遵守)

| 档位 | 关键词 | 用户画像 |
|---|---|---|
| **0 分** | 不了解 / 没接触 / 没试过 | 观望者 |
| **1 分** | 听过 / 试过 / 玩过 | 入门 |
| **2 分** | 能用 / 知道 / 有基础 | 实战 |
| **3 分** | 体系 / SOP / 团队复用 / 可输出 | 顶端 |

**反例(不要写)**:
- ❌ 选项之间是平行选择(用 Claude / 用 GPT / 用豆包)—— 这考的是偏好不是水平
- ❌ 选项之间没拉开差距(经常用 / 偶尔用 / 有时用)—— 用户分不出来
- ❌ 选项太技术化(知道 RAG / 用过 LangChain)—— 老板/专家答不了

## 3. 月度选题:从哪里抓 AI 趋势

每月初花 15 分钟扫这几个源,挑出 4-5 个**已落地、有用户感知**的趋势:

| 信息源 | 重点看 | 频次 |
|---|---|---|
| Anthropic / OpenAI / Google 官方博客 | 新模型发布、新能力 | 月初 |
| Hacker News 月度热榜 | 国外开发者实际在用什么 | 月初 |
| X (Twitter) AI KOL 列表 | @sama @karpathy @swyx 等 | 周扫 |
| 量子位 / 机器之心 | 国内动态、国产模型 | 周扫 |
| Product Hunt 月度榜 | AI 产品商业化进展 | 月初 |
| a16z / Sequoia AI 文章 | 资本视角的趋势判断 | 季度 |

### 选题筛选标准(只选满足全部 3 条的)

1. ✅ **已经能用** — 不是论文/期货,有产品或 API 可访问
2. ✅ **用户感知得到** — 行业老板/专家在饭局上能聊到
3. ✅ **3 个月内不会过时** — 不是短命噱头(如某个一周热度的小工具)

> 例子(2026.05 入选):Sora 2 / Computer Use / MCP 协议 / Extended Thinking / 多 Agent 编排
> 例子(应淘汰):某个小众插件的具体用法、某次模型小版本号更新

## 4. 替换 vs 新增的取舍

**保持总题数 20 不变**(每维 4 题),用新题**替换**旧题,而不是无限累加。

| 替换标准 | 例子 |
|---|---|
| 旧题描述的能力已成基础 → 换前沿题 | "听过 AI Agent 吗" 在 2026 已不分层,可换 "用过 Computer Use 吗" |
| 工具名过时 → 换当下主流 | "WorkBuddy" 已迭代,换为当下中文 Agent 旗舰 |
| 选项区分度变低 → 重设梯度 | 选 3(深度用户)的人超过 30% → 把梯度往上拉 |

### 何时该新增维度?

当前 5 维稳定。**只有出现以下情况才考虑加第 6 维**:
- 出现了完全无法归入现有 5 维的新能力(如"AI 硬件交互"成为主流)
- 5 个维度之外的某项能力已能独立预测用户付费意愿

## 5. 示例:2026.05 → 2026.06 一次完整更新

假设 6 月新趋势是 **GPT-5o 多模态实时交互** + **Claude Agent SDK 普及**,更新如下:

```diff
- const QBANK_VERSION = '2026.05';
- const QBANK_UPDATED = '2026-05-15';
+ const QBANK_VERSION = '2026.06';
+ const QBANK_UPDATED = '2026-06-10';

  // 维度0 工具认知 —— 替换第 4 题
- { d:0, q:'2026 年第一梯队模型(Claude 4.x / GPT-5 / Gemini 2.x / 国产顶级模型)你怎么看?', a:[...] },
+ { d:0, q:'GPT-5o 等多模态实时交互模型(语音+视觉+实时打断)你的使用程度?',
+   a:['没用过','试过几次','日常会用语音+视觉模式','已纳入工作流,会议/教学/客户演示常用'] },

  // 维度3 自动化思维 —— 替换 Computer Use 那题
- { d:3, q:'对「Computer Use」/「Browser Agent」...',
+ { d:3, q:'Claude Agent SDK / OpenAI Agents SDK 这类「Agent 开发框架」你的实践?',
+   a:['没听过','看过 demo','跑过官方 sample','已用它给团队定制了内部 Agent'] },
```

只换 2 道,保留另外 3 道前沿题。**最小更新成本 = 最大跟新效果**。

## 6. 质检清单(发版前 5 分钟自查)

- [ ] 20 道题、5 维各 4 道(在浏览器开发者工具跑一遍 `console.table(QS.reduce((acc,q)=>{acc[q.d]=(acc[q.d]||0)+1;return acc},{}))`)
- [ ] 每题 4 个选项,梯度从 0 分到 3 分递进
- [ ] 选项里没有专有技术名词无解释(如 `RAG` `embedding` 这种要避免或加注)
- [ ] 自己跑一遍测评,各档位都试一下,看是否能稳定落到对应段位
- [ ] 翻一遍精灵 mock 回复(`mockReply` 函数),把过时的工具名/版本号顺手更新

## 7. 后续工程化方向(当前先不做)

题库稳定每月迭代后,有 ROI 时再考虑:

1. **题库外置 JSON** — 把 `QS` 抽到 `/qbank/2026-06.json`,前端 `fetch` 加载
2. **多版本并存** — 老用户可重测旧版本,看自己在同一道题上的进化
3. **管理后台** — 不改代码就能改题(Notion DB → 自动生成 JSON 是性价比最高的路径)
4. **AB 测试** — 同一道题写 2 个版本,看哪个区分度更好
5. **题目命中率分析** — 记录每个选项的选择分布,选 0 分或选 3 分的人 >70% 的题应淘汰

## 8. 应急回滚

如果某月题目改完发现区分度变差(用户得分集中扎堆),立即回滚:

```bash
git log --oneline -- website/index.html | head -5
git checkout <上一版本 commit> -- website/index.html
```

或者保留旧题库的备份文件:`website/qbank-archive/2026-05.html`,直接复制 `QS` 数组覆盖回去。

---

**最后一条提醒**:题库的核心价值不是覆盖最新名词,而是**让用户准确地认识自己**。
能让一个用了 AI 三年的专家答完后说"原来我的瓶颈在这",比把所有 2026 前沿词堆进去更有价值。
